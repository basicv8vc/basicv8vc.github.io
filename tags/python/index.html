使用pdb调试Python代码  [draft]

一直使用PyCharm写Python，最近需要在服务器调试项目代码，Google了Python内置模块pdb的基本用法，这里记录一下。 假设项目代码是： # code.py a = "aaa" b = "bbb" c = "ccc" def combine(s1, s2): s3 = s1 + s2 + s1 s4 = s3 * 2 return s4 d = combine(a, b) final = a + b + c + d print final 开启pdb 在Python代码中使用pdb有两种方式： 源文件中导入pdb模块，在需要设置断点(breakpoint)的地方添加语句:pdb.set_trace()，然后在系统终端执行 $ python code.py 在系统终端执行 $ python -m pdb code.py, 然后再使用pdb创建断点命令设置断点。 我个人更推荐第二种方式，原因是不需要修改源代码。 pdb命令 在终端执行, $ python -m pdb code.py 进入调试模式，首先看到-> a = "aaa"，它的意思是接下来要执行的语句是a = "aaa"。...
